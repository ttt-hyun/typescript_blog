# 📁함수형 컴포넌트 (Functional Components)

## 📄 기본 형태
```javascript
const page = () => {
  return (
    <div>
      <h1>Hello, world!</h1>
    </div>
  );
}
```
## 📄 특징
1. 클래스형 컴포넌트보다 메모리가 덜 사용된다.
2. 단순한 구조로 사용이 편하다.
3. React와 Next 공식 문서에서 권장하고 있는 컴포넌트 방식이다.

\* 메모리가 덜 사용되는 이유는 클래스형 컴포넌트에서 사용되는 생명주기 메서드(Hooks)들과 this 바인딩 과정을 생략하여 사용이 가능하기 때문이다.

# 📁클래스형 컴포넌트 (Functional Components)

## 📄 기본 형태
```javascript
import React, { Component } from 'react';
class Page extends Component {
  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
      </div>
    );
  }
}
```
## 📄 특징
1. 함수형 컴포넌트보다 메모리가 더 사용된다.
2. componentDidMount와 componentWillUnmount 등과 같은 생명주기 메서드들이 존재한다.
3. 컴포넌트를 사용하기 위해 react의 Component를 반드시 import 해줘야 한다.
