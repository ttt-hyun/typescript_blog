# 제너릭(Generic)이란?

## 정의

제너릭이란 타입의 일반화를 의미  
타입의 일반화란 클래스나 함수를 만들 때 타입을 지정하는 것이 아닌,  
사용할 때 타입을 지정할 수 있게 하는 것

## 사용이유

1. 재사용성  
클래스나 함수가 특정 타입에 묵여있지 않아 재사용성을 높인다.
2. 오류처리  
제너릭을 사용하지 않는다면 `any`를 사용하게 될텐데 해당 타입은 타입유형을 검사하지 않기  
때문에 런타임 단계에서 오류를 이르킬 가능성이 커진다. 하지만 제너릭을 사용하게 된다면 런타임에러를  
줄일 수 있다.

# 제너릭 사용하기

## 기본 사용법

```js
function <T>doIt(a: T): T{
    return a;
}
// 함수 사용
const work = doIt<string>("make string");
```
제너릭을 사용하는 기본 문법이다. 꺽쇠(angle brackets)에 변수를 넣어(자유롭게 넣을 수 있으나 네이밍 컨벤션으로 T를 사용) 
유저가 준 인수의 타입을 캡처하여 이 정보를 나중에 사용할 수 있게 한다.  
함수 사용에서 보면 함수에서 `string` 인수를 통해 모든 인자 `T`에 전달하고 있다.

## 타입 인수 추론

컴파일러는 우리가 작성한 코드를 토대로 인수를 추론하여 제너릭으로 사용 가능하게 해준다.

```js
function <T>doIt(a: T): T{
    return a;
}

const work = doIt("make string");
```
위의 코드는 함수에 인수를 전달하지 않았다. 하지만 컴파일러는 우리가 전달한 인수에 따라
인자 `T`의 값을 정하게 된다. 더욱 코드를 간결하고 가독성 있게 작성 할 수 있는 것이다.
